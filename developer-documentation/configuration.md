# Configuration

####

| Properties                                            | Description                                                                                                                                                                                                                                                                                                                                    |
|-------------------------------------------------------| ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| <h4>notification_enabled</h4>                         | boolean value which determines whether notification is to be sent or not                                                                                                                                                                                                                                                                       |
| <h4>notification\_url</h4>                            | url which will be used to send notification url                                                                                                                                                                                                                                                                                                |
| <h4>registry\_perRequest\_indexCreation\_enabled</h4> | boolean value which determines if index needs to be created at database level                                                                                                                                                                                                                                                                  |
| <h4>external\_entities</h4>                           | comma separated strings of external entities                                                                                                                                                                                                                                                                                                   |
| <h4>workflow\*\*.\*\*enable</h4>                      | boolean value to enable or disable the workflow i.e attestation policy                                                                                                                                                                                                                                                                         |
| <h4>async\_enabled</h4>                               | create entities asynchronously                                                                                                                                                                                                                                                                                                                 |
| <h4>kafka\_create\_entity\_topic</h4>                 | if async\_enabled, then this property defines the topic name to which new async requests are pushed                                                                                                                                                                                                                                            |
| <h4>kafka\_post\_create\_entity\_topic</h4>           | if async is enabled, then this property defines the topic name to which the response of create-entity is pushed                                                                                                                                                                                                                                |
| <h4>search\_offset</h4>                               | this is the default offset that will be used while searching,                                                                                                                                                                                                                                                                                  |
| <h4>search\_limit</h4>                                | this is the default limit that will be used or the max value that can be used to as a limit in the search                                                                                                                                                                                                                                      |
| <h4>search\_expandInternal</h4>                       | boolean value, if set true, all the search results will be expanded to include internal objects.                                                                                                                                                                                                                                               |
| <h4>database\_provider</h4>                           | this property states which database is to be used. Providers available are NEO4J, SQLG, CASSANDRA, ORIENTDB, TINKERGRAPH (in-memory)                                                                                                                                                                                                           |
| <h4>connectionInfo\_uri</h4>                          | url to connect to database. eg Postgres - jdbc:postgresql://localhost:5432/yourdb                                                                                                                                                                                                                                                              |
| <h4>connectionInfo\_username</h4>                     | username to be used to connect to database                                                                                                                                                                                                                                                                                                     |
| <h4>connectionInfo\_password</h4>                     | password to be used to connect to database                                                                                                                                                                                                                                                                                                     |
| <h4>connectionInfo\_maxPoolSize</h4>                  | Database connection pool size                                                                                                                                                                                                                                                                                                                  |
| <h4>verify\_url</h4>                                  | this url will be used to verify the issued VC. point it to {certification\_signer}/verify                                                                                                                                                                                                                                                      |
| <h4>certificate\_health\_check\_url</h4>              | certificate api service's health check url                                                                                                                                                                                                                                                                                                     |
| <h4>template\_folder\_path</h4>                       | path from which default template will be loaded if no external template is provided. File name should be the **`name_of_schema.appropriate_extention`** The location should be in resources folder                                                                                                                                             |
| <h4>audit\_enabled</h4>                               | This audit configuration will enable audit logging in the system                                                                                                                                                                                                                                                                               |
| <h4>audit\_frame\_store</h4>                          | <p>FILE : Store the audit log in files.<br>DATABASE : Store the audit log in primary database configured in database properties<br>ELASTIC : Store the audit log in elastic search configured in elastic properties.</p>                                                                                                                       |
| <h4>audit\_suffix</h4>                                | suffix which is used for schema name to store audit information for a particular entity type                                                                                                                                                                                                                                                   |
| <h4>audit\_suffixSeparator</h4>                       | separator between entity name and audit\_suffix                                                                                                                                                                                                                                                                                                |
| <h4>keycloack\_user\_email\_actions</h4>              | email actions which will be trigger by keycloak example email actions: VERIFY\_EMAIL, UPDATE\_PROFILE, UPDATE\_PASSWORD,TERMS\_AND\_CONDITIONS etc. email details should be configured in keycloak realm settings                                                                                                                              |
| <h4>validation\_enabled</h4>                          | boolean property to check for request body to be validated in specific format                                                                                                                                                                                                                                                                  |
| <h4>validation\_type</h4>                             | type in which request body needs to be validated. eg JSON                                                                                                                                                                                                                                                                                      |
| <h4>service\_connection\_timeout</h4>                 | Set the connection timeout in milliseconds for the underlying request configuration                                                                                                                                                                                                                                                            |
| <h4>service\_connection\_request\_timeout</h4>        | Set the timeout in milliseconds used when requesting a connection from the connection manager using the underlying request Configuration                                                                                                                                                                                                       |
| <h4>service\_read\_timeout</h4>                       | Set the socket read timeout in milliseconds for the underlying request configuration                                                                                                                                                                                                                                                           |
| <h4>http\_max\_connections</h4>                       | maximum http connections                                                                                                                                                                                                                                                                                                                       |
| <h4>taskExecutor\_index\_threadPoolName</h4>          | Specify the prefix to use for the names of newly created threads.                                                                                                                                                                                                                                                                              |
| <h4>taskExecutor\_index\_corePoolSize</h4>            | Set the ThreadPoolExecutor's core pool size.                                                                                                                                                                                                                                                                                                   |
| <h4>taskExecutor\_index\_maxPoolSize</h4>             | Set the ThreadPoolExecutor's maximum pool size.                                                                                                                                                                                                                                                                                                |
| <h4>taskExecutor\_index\_queueCapacity</h4>           | Set the capacity for the ThreadPoolExecutor's BlockingQueue.                                                                                                                                                                                                                                                                                   |
| <h4>auditTaskExecutor\_threadPoolName</h4>            | Specify the prefix to use for the names of newly created threads.                                                                                                                                                                                                                                                                              |
| <h4>auditTaskExecutor\_corePoolSize</h4>              | Set the ThreadPoolExecutor's core pool size.                                                                                                                                                                                                                                                                                                   |
| <h4>auditTaskExecutor\_maxPoolSize</h4>               | Set the ThreadPoolExecutor's maximum pool size.                                                                                                                                                                                                                                                                                                |
| <h4>auditTaskExecutor\_queueCapacity</h4>             | Set the capacity for the ThreadPoolExecutor's BlockingQueue.                                                                                                                                                                                                                                                                                   |
| <h4>elastic\_search\_connection\_url</h4>             | url for elastic search connection if elastic search is enabled. Elastic search is enabled using search\_providerName                                                                                                                                                                                                                           |
| <h4>search\_providerName</h4>                         | The search mechanism to use. Values could be either NativeSearchService or ElasticSearchService. If NativeSearchService, then every search API uses the same database as the writes. May not offer high speed reads. This is the default search service, if this config is not provided. If ElasticSearchService, then Elastic search is used. |
| <h4>sunbird\_sso\_realm</h4>                          | keycloak realm name to be used for authentication and authorization                                                                                                                                                                                                                                                                            |
| <h4>sunbird\_sso\_url</h4>                            | keycloak connection url                                                                                                                                                                                                                                                                                                                        |
| <h4>sunbird\_sso\_admin\_client\_id</h4>              | client id to be used as admin                                                                                                                                                                                                                                                                                                                  |
| <h4>sunbird\_sso\_admin\_client\_secret</h4>          | secret key of keycloak admin client set by using sunbird\_sso\_admin\_client\_id                                                                                                                                                                                                                                                               |
| <h4>claims\_url</h4>                                  | url in order to connect to claims service                                                                                                                                                                                                                                                                                                      |
| <h4>sign\_url</h4>                                    | url in order to connect to certificate signer to sign the document                                                                                                                                                                                                                                                                             |
| <h4>sign\_health\_check\_url</h4>                     | certificate signer health check url                                                                                                                                                                                                                                                                                                            |
| <h4>signature\_enabled</h4>                           | boolean value which determines whether signature is to be created for a document                                                                                                                                                                                                                                                               |
| <h4>pdf\_url</h4>                                     | url to fetch certificate from certificate api service                                                                                                                                                                                                                                                                                          |
| <h4>certificate\_health\_check\_url</h4>              | certificate api health check url                                                                                                                                                                                                                                                                                                               |
| <h4>template\_base\_url</h4>                          | api endpoint to get default templates stored in sunbird-rc                                                                                                                                                                                                                                                                                     |
| <h4>sunbird\_keycloak\_user\_set\_password</h4>       | boolean value to default password for user/owner of entity in keycloak                                                                                                                                                                                                                                                                         |
| <h4>filestorage\_connection\_url</h4>                 | minio connection url to store files                                                                                                                                                                                                                                                                                                            |
| <h4>filestorage\_access\_key</h4>                     | access key of minio                                                                                                                                                                                                                                                                                                                            |
| <h4>filestorage\_secret\_key</h4>                     | secret key of minio                                                                                                                                                                                                                                                                                                                            |
| <h4>filestorage\_bucket\_key</h4>                     | bucket name where to store the files in minio                                                                                                                                                                                                                                                                                                  |
| <h4>registry\_base\_apis\_enable</h4>                 | if enabled the exisitng /add /update apis will be enabled. This is to enable backward compatibility                                                                                                                                                                                                                                            |
| <h4>sunbird\_keycloak\_user\_password</h4>            | if sunbird\_keycloak\_user\_set\_password is set to true, provide this value to set this as default user password                                                                                                                                                                                                                              |
| <h4>logging.level.root</h4>                           | <p>The log level that will be used for logging. Default logging is INFO.<br>Values supported:<br>INFO<br>DEBUG<br>WARN<br>ERROR</p>                                                                                                                                                                                                            |
| <h4>enable\_external\_templates</h4>                  | boolean value which when set to true, one can retrieve the certificate pdf using external templates                                                                                                                                                                                                                                            |
| <h4>authentication\_enabled</h4>                      | boolean value to enable authentication in the system                                                                                                                                                                                                                                                                                           |
| <h4>kafka\_bootstrap\_address</h4>                    | url for kafka connection                                                                                                                                                                                                                                                                                                                       |
| <h4>webhook\_enabled</h4>                             | boolean value to enable webhook if async\_enabled is true                                                                                                                                                                                                                                                                                      |
| <h4>webhook\_url</h4>                                 | if async\_enabled and webhook\_enabled is set to true, the caller can retrieve information of the created entity once it is generated via this webhook url                                                                                                                                                                                     |
| <h4>redis\_host</h4>                                  | redis connection url                                                                                                                                                                                                                                                                                                                           |
| <h4>redis\_port</h4>                                  | port on which redis is running                                                                                                                                                                                                                                                                                                                 |
| <h4>manager\_type</h4>                                | if using a single instance of registry, set this value to DefinitionsManager else set to DistributedDefinitionsManager                                                                                                                                                                                                                         |
| <h4>service\_retry\_maxAttempts</h4>                  | The number of times an attempt must be made to reach to the service                                                                                                                                                                                                                                                                            |
| <h4>service\_retry\_backoff\_delay</h4>               | The fixed time interval, in milliseconds, between each such attempt.                                                                                                                                                                                                                                                                           |

### Notification Service

| Properties                                                                           | Description                                                                               |
|--------------------------------------------------------------------------------------| ----------------------------------------------------------------------------------------- |
| <h4>KAFKA\_BOOTSTRAP\_SERVERS in environment variable or kafka.bootstrapServers</h4> | host along with port if port is different than default i.e 9092 on which kafka is present |
| <h4>kafka.notifyTopic</h4>                                                           | topic name which holds notification messages                                              |
| <h4>smsapi.url</h4>                                                                  | api url to send sms to provided mobile numbers                                            |
| <h4>smsapi.authKey</h4>                                                              | authorization key for sms api url                                                         |
| <h4>smsapi.enable</h4>                                                               | flag to enable sending sms using sms api                                                  |
| <h4>smsapi.requestTemplate</h4>                                                      | message template to be sent to mobile number in notification message                      |
| <h4>emailsmtp.fromAddress</h4>                                                       | senders email address from which to send email                                            |
| <h4>emailsmtp.password</h4>                                                          | password for that email address                                                           |
| <h4>emailsmtp.enable</h4>                                                            | flag to enable sending email using email api                                              |
| <h4>TRACK\_NOTIFICATIONS in environment variable</h4>                                | boolean value to track all sent notifications and fetch them if needed                    |
